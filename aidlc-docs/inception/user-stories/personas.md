# 페르소나

## 페르소나 1: 고객

- **역할**: 테이블 태블릿에서 직접 주문하는 매장 손님.
- **주요 목표**: 메뉴를 빠르게 탐색하고 주문하며, 현재 테이블의 주문 내역을 직원 도움 없이 확인한다.
- **상황**: 테이블에 놓인 브라우저 기반 태블릿을 사용한다. 별도 수동 로그인은 원하지 않는다.
- **필요**:
  - 테이블 자동 로그인을 통한 즉시 접근.
  - 명확한 메뉴 카테고리와 가격.
  - 단순한 장바구니 수량 조작.
  - 주문 성공 여부에 대한 확신.
  - 현재 세션 주문 내역만 조회.
- **불편 요소**:
  - 복잡한 메뉴 탐색.
  - 새로고침 또는 주문 실패 후 장바구니 유실.
  - 이전 손님의 주문 내역 노출.

## 페르소나 2: 매장 관리자

- **역할**: 주문, 테이블, 메뉴를 관리하는 매장 운영자.
- **주요 목표**: 들어오는 주문을 실시간으로 확인하고 영업 중 테이블 세션을 관리한다.
- **상황**: 매장 운영 중 브라우저 기반 관리자 대시보드를 사용한다.
- **필요**:
  - 단순한 관리자 대시보드 접근.
  - 테이블별 신규 주문 실시간 가시성.
  - 주문 상태 변경.
  - 확인 후 잘못된 주문 삭제.
  - 테이블 이용 완료 처리와 이력 보존.
  - 기본 메뉴 관리.
- **불편 요소**:
  - 주문 누락 또는 지연 표시.
  - 새 손님을 위한 테이블 초기화 어려움.
  - 테이블 완료 후 과거 주문 기록 유실.

## 페르소나와 스토리 매핑

| 페르소나 | 스토리 그룹 |
|---|---|
| 고객 | 테이블 설정, 메뉴 탐색, 장바구니 관리, 주문 생성, 현재 세션 주문 내역 |
| 매장 관리자 | 관리자 로그인, 실시간 주문 모니터링, 테이블 관리, 과거 주문 조회, 메뉴 관리 |
